关于16系显卡无法运行yolo_v8的解决方案

唐可爱 happy
  2023-02-07 20:02:42 2023-02-07 20:02  
  • 深度学习 
 

问题描述

本人笔记本配置CPU:I9-9300H，GPU：GTX1660Ti，在使用yolo_v8进行深度学习的过程中发现在训练loss计算一直是Nan，起初考虑过是不是学习率太大导致，后来降低学习率，情况并没有改善，并且仅训练一个epoch时，loss计算仍为Nan。

百度寻找解决方案，发现降低cuda版本为102，可以解决问题，但是Pytorch官方已经逐渐不维护10.x版本的cuda，所以另寻其他方法。

查询NVIDIA官方网站，发现16系显卡没有Tensor core核心，不支持FP16半精度数据类型。

解决方案

通过阅读源码，yolo_v8版本在使用显卡训练是会使用FP16进行计算。因此只需要在判断是否是GPU训练时，将true改为false即可。

更改位置如下：

  1. ultralytics\yolo\engine\trainer.py 第117行，改为self.amp = False

  2. ultralytics\yolo\engine\validator.py 第86行，改为self.args.half = False

  3. ultralytics\yolo\engine\validator.py 第98行，改为self.args.half = False

结尾

经过实际测试，该方法可以解决yolo_v8代码在16系显卡上运行的问题，同时该方法也可运用于其他同样无Tensor core核心的显卡。
